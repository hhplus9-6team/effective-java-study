# item 59: 라이브러리를 익히고 사용하라

> [!IMPORTANT]
> 이미 있는 표준 라이브러리를 먼저 확인하라.  
> 검증된 코드와 성능을 활용하면 버그를 줄이고 유지보수를 쉽게 한다. 직접 구현은 최후의 수단이다.

## 이점
### 1. 이미 잘 만들어진 라이브러리가 존재한다면 직접 구현하지 말라
- 자바에는 이미 수많은 기능이 표준 라이브러리에 안정적으로 제공된다
- 이런 라이브러리는 많은 개발자들이 수년간 사용하면서 버그가 드러나고 개선된 검증된 코드다
- 직접 구현하면 버그 발생 확률이 높고, 성능도 검증되지 않는다

#### 근거
Math.random() 대신 직접 난수 생성 알고리즘을 구현하지만, 테스트해보면 기대와 달리 분포가 일정하지 않고 특정 숫자에 몰리는 현상이 발견된다  
→ java.util.Random 또는 ThreadLocalRandom, 더 나아가 SecureRandom 같은 표준 라이브러리를 사용하는 것이 훨씬 안전하고 성능도 좋다!

### 2. 핵심적인 일과 크게 관련없는 문제를 해결하느라 시간을 낭비하지 않아도 된다.

### 3. 성능이 지속적으로 개선된다
- 사용자가 많고 업게 표준 벤치마크를 사용해 성능을 확인하기 떄문에 꾸준히 더 나은 방법을 모색할 수 밖에 없다
- 예를 들어, 새로운 JDK가 나오면 내부 최적화나 버그 수정이 이루어지므로 라이브러리를 쓰기만 해도 성능이 자동으로 개선되는 효과를 얻을 수 있다


### 4. 기능이 점점 많아진다
- 부족한 부분이 있다면 개발자 커뮤니티에서 이야기가 나오고 다음 릴리스때 해당 기능이 추가되곤 한다

### 5. 작성한 코드가 많은 사람에게 낯익은 코드가 된다
- 자연스럽게 다른 개발자도 쉽게 이해하고 유지보수가 좋고 재활용하기 쉬운 코드가 된다
- 직접 만든 유틸은 문서화도 부족하고, 팀원의 지식 편차로 인해 유지보수가 어려워진다

## 하지만 실상 많은 프로그래머들은 직접 구현해서 쓰고있다. 왜?
### 표준 라이브러리를 잘 모른다
- 표준 라이브러리에 어떤 기능이 있는지, 어떻게 사용하는지 잘 모르는 경우가 많다
- 메이저 릴리스마다 주목할만한 기능이 추가 되고, 웹페이지에 공시해주니 읽어볼 만하다


## 자바 개발자라면 반드시 숙지해야 할 주요 라이브러리들

### - Java 표준 라이브러리 (특히: java.lang, java.util, java.io와 그 하위 패키지)
### - 컬렉션 프레이뭐크와 스트림 라이브러리
### - java.util.concurrent)
- 멀티스레드 프로그래밍 작업을 단순화 해주는 고수준 편의 기능 부터 자신만이 고수준 개념을 직접 구현할 수 있는 저수준 요소까지 제공한다
