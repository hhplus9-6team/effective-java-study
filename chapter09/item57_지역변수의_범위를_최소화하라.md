# ITEM 57. 지역변수의 범위를 최소화하라

## 지역변수 범위를 최소화해야 하는 이유

- 지역변수의 유효 범위를 최소로 줄이면 코드 가독성과 유지보수성이 높아지고 오류 가능성은 낮아진다
- 지역변수의 범위가 넓으면 변수가 사용되는 곳을 모두 추적해야 하므로 코드 이해가 어려워진다

## 지역변수 범위를 최소화하는 방법

### 1. 가장 처음 쓰일 때 선언하기

- 지역변수를 사용하기 전에 선언하는 것이 아니라, **가장 처음 쓰일 때 선언**해야 한다
- 변수를 사용하기 훨씬 전에 선언하면 코드 가독성이 떨어지진다

### 2. 거의 모든 지역변수는 선언과 동시에 초기화해야 한다

**예외:**
- try-catch 문에서 초기화해야 하는 경우
- 변수를 초기화하는 표현식에서 검사 예외를 던질 수 있는 경우

### 3. while 문보다는 for 문을 사용하라

**while 문의 문제점:**
```java
Iterator<Element> i = c.iterator();
while (i.hasNext()) {
    doSomething(i.next());
}

Iterator<Element> i2 = c2.iterator();
while (i.hasNext()) {  // 버그! i2가 아니라 i를 사용
    doSomething(i2.next());
}
```

**for 문의 장점:**
```java
for (Iterator<Element> i = c.iterator(); i.hasNext(); ) {
    Element e = i.next();
    // ...
}

// 다음 코드에서 i를 사용하려고 하면 컴파일 에러 발생
for (Iterator<Element> i2 = c2.iterator(); i.hasNext(); ) {
    Element e2 = i2.next();
    // ...
}
```

**for 문이 더 나은 이유:**
- 반복문 변수의 범위가 반복문의 몸체와 괄호 안으로 제한됨
- 변수 이름을 실수로 재사용할 가능성이 줄어듦
- 더 짧아서 가독성이 좋음

### 4. 메서드를 작게 유지하고 한 가지 기능에 집중하라

- 한 메서드에서 여러 기능을 처리하면 지역변수가 여러 기능에 걸쳐 사용될 수 있음
- 메서드를 기능별로 분리하면 자연스럽게 지역변수 범위가 최소화됨

## 결론

- **지역변수는 가장 처음 쓰일 때 선언하라**
- **거의 모든 지역변수는 선언과 동시에 초기화하라**
- **while 문보다는 for 문을 사용하라**
- **메서드를 작게 유지하고 한 가지 기능에 집중하라**
