# item77. 예외를 무시하지 말라

API 설계자가 메서드 선언에 예외를 명시한 이유는, 그 메서드를 사용할 때 적절한 조치를 취해달라는 뜻이다. catch 블록을 비워두면 예외가 존재할 이유가 없어진다.

---

## catch 블록을 비워두면 예외가 존재할 이유가 없어진다

```java
try {
    // ...
} catch (SomeException e) {
}
```
화재 경보를 무시하는 것

---

## 예외를 무시해야 하는 경우

`FileInputStream`을 닫을 때 
- 읽기전용 스트림이여서 파일의 상태를 변경하지 않았으므로 복구할 것이 없다
- 스트림을 닫는다는 것은 필요한 정보를 이미 다 읽었다는 뜻

이런 경우에도 파일을 닫지 못했다는 사실을 로그로 남기는 것이 좋다.

예외를 무시하기로 결정했다면 `catch` 블록 안에 그렇게 한 이유를 주석으로 남기고, 예외 변수의 이름도 `ignored`로 바꾸자

```java
} catch (IOException ignored) {
    // 사유작성
}
```

Q1. 예외를 무시하면 안 되는 이유는 무엇인가요?

Q2. 예외를 무시해도 되는 경우는 무엇인가요?
